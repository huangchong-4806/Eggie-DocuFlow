# Eggie DocuFlow V1.3.6 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将当前已确认的 OCR、全局进度、PDF 批量转换、页面整理、批量改名和数量保护功能打包为 Eggie DocuFlow V1.3.6，通过本地和线上验证后发布到 GitHub。

**Architecture:** 先将当前确认的代码保存为可回退节点，再同步线上 `main` 并建立 `codex/release-v1.3.6` 分支。版本号以 `version.py` 为单一来源，README、macOS 安装包信息和发行说明保持一致。只有最终 ZIP 通过启动、签名、完整性、敏感文件和校验值检查后才允许公开发布。

**Tech Stack:** Python 3.9/3.10, PySide6, PyInstaller, macOS codesign, unittest, GitHub Actions, GitHub Release

## Global Constraints

- 正式版本号为 `1.3.6`，GitHub 标签为 `v1.3.6`。
- 包名为 `EggieDocuFlow_V1.3.6_mac.app` 和 `EggieDocuFlow_V1.3.6_mac.zip`。
- 仅支持 Apple 芯片 Mac，macOS 11.0 及以上，未做 Apple 公证。
- 不恢复 45 MB 自动拦截，只记录安装包大小。
- 不发布 `.env`、密钥、真实业务资料、本机日志、旧安装包、旧截图、参考稿、缓存或临时文件。
- V1.3.5 发布页和附件保持不变。
- 在真正点击公开发布前，向用户报告最终大小和 SHA-256 并再次确认。
